Comandos Básicos do Docker

docker run

docker run [opções] imagem [comando]

Cria e inicia um container. Por exemplo, docker run hello-world executa um container que imprime uma mensagem de boas-vindas.

Exemplos:

docker run hello-world Executa container de teste
docker run -it python bash Executa Python interativamente
docker run -d nginx Executa Nginx em background

docker pull

docker pull nome-da-imagem[:tag]

Baixa uma imagem Docker do Docker Hub ou de outro registro. Por exemplo, docker pull ubuntu baixa a imagem mais recente do Ubuntu.

Exemplos:

docker pull ubuntu Baixa última versão do Ubuntu
docker pull python:3.9 Baixa Python versão 3.9

docker build

docker build -t nome-imagem:tag .

Constrói uma imagem Docker a partir de um Dockerfile. Este comando é fundamental no processo de criação de imagens personalizadas.

Exemplos:

docker build -t minha-app:1.0 . Constrói imagem com tag
docker build -f Dockerfile.prod -t app:prod . Usa Dockerfile específico

Gerenciamento de Containers

docker ps

docker ps [opções]

Lista todos os containers em execução. Com opções adicionais, também é possível ver containers parados.

Exemplos:

docker ps Lista containers em execução
docker ps -a Lista todos os containers (inclusive parados)
docker ps -q Lista apenas IDs dos containers

docker start

docker start <id-ou-nome>

Inicia um container parado.

Exemplos:

docker start meu-container Inicia container pelo nome
docker start a1b2c3d4 Inicia container pelo ID

docker stop

docker stop <id-ou-nome>

Para um container em execução, seguido do ID ou nome do container.

Exemplos:

docker stop meu-container Para container pelo nome
docker stop $(docker ps -q) Para todos os containers em execução

docker rm

docker rm <id-ou-nome>

Remove um ou mais containers. É importante garantir que o container esteja parado antes de tentar removê-lo.

Exemplos:

docker rm meu-container Remove container
docker rm -f meu-container Força remoção de container em execução
docker rm $(docker ps -aq) Remove todos os containers parados

docker exec

docker exec -it <id> bash

Entra no terminal (bash) de um container em execução. Permite executar comandos dentro do container.

Exemplos:

docker exec -it meu-container bash Acessa bash do container
docker exec meu-container ls -la Executa comando específico
docker exec -it meu-container sh Acessa shell (para Alpine Linux)

Trabalhando com Imagens Docker

docker images

docker images

Lista todas as imagens Docker disponíveis localmente.

Exemplos:

docker images Lista todas as imagens
docker images -q Lista apenas IDs das imagens
docker images --filter "dangling=true" Lista imagens órfãs

docker rmi

docker rmi <id-da-imagem>

Remove uma imagem local, seguido do ID da imagem.

Exemplos:

docker rmi ubuntu:20.04 Remove imagem específica
docker rmi -f a1b2c3d4 Força remoção da imagem
docker rmi $(docker images -q) Remove todas as imagens

docker tag

docker tag origem:tag destino:tag

Cria uma tag para uma imagem existente.

Exemplos:

docker tag minha-app:latest minha-app:v1.0 Adiciona nova tag

docker push

docker push usuario/imagem:tag

Envia uma imagem para o Docker Hub ou outro registry.

⚠️ Importante: É necessário estar autenticado no Docker Hub para fazer push. Use docker login antes de executar este comando.

Exemplos:

docker login Faz login no Docker Hub
docker push otavioustra/minha-app:1.0 Envia imagem para o registry

🎯 Práticas do Mini-Curso

Prática 01

Rodando Python no Docker

Objetivo: Executar código Python dentro de um container Docker sem precisar instalar Python localmente.

📝 Passo a Passo:

1

Rode a imagem do Python usando:

docker run -it python bash

Este comando baixa a imagem do Python e abre o bash interativo.

2

Dentro do bash do container, rode o interpretador do Python:

python
3

Escreva um código em Python. Exemplo:

print("Hello, container!")
4

Saia do interpretador Python:

exit()
5

Saia do bash do container:

exit

✨ Dica: Você pode usar Ctrl + D como atalho para sair do Python ou do bash.

Prática 02

Dockerizando uma Aplicação Python

Objetivo: Criar e executar uma aplicação Python containerizada que busca piadas de uma API.

📁 Arquivos da aplicação: Baixar do Google Drive

📝 Passo a Passo:

1

Criar um Dockerfile:

Crie um arquivo chamado Dockerfile (sem extensão) com o seguinte conteúdo:

FROM python:3.10-slim WORKDIR /app COPY requirements.txt . COPY main.py . RUN pip install --no-cache-dir -r requirements.txt CMD ["python", "main.py"]
2

Criar o arquivo da aplicação (main.py):

import requests def get_joke(): url = "https://official-joke-api.appspot.com/random_joke" response = requests.get(url) if response.status_code == 200: joke = response.json() print(f"{joke['setup']} ... {joke['punchline']}") else: print("Não foi possível obter uma piada.") if __name__ == "__main__": get_joke()

Código disponível na pasta APP_Piadas do projeto.

3

Criar o arquivo requirements.txt:

requests
4

Criar uma imagem a partir do Dockerfile:

docker build -t app-piadas .

Este comando constrói a imagem Docker com o nome "app-piadas".

5

Criar conta no Docker Hub e publicar o repositório com a imagem:

docker login

Faça login com suas credenciais do Docker Hub.


docker tag app-piadas seu-usuario/app-piadas:1.0

Substitua "seu-usuario" pelo seu username do Docker Hub.


docker push seu-usuario/app-piadas:1.0

Publica a imagem no Docker Hub.

6

Executar a imagem guardada no Docker Hub:

docker run seu-usuario/app-piadas:1.0

Executa o container e exibe uma piada aleatória!

✨ Dica: Você pode executar várias vezes para obter piadas diferentes! Use docker run app-piadas localmente se já tiver a imagem construída.
